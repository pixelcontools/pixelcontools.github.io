Getting Started

An introduction to PixelConnect — a pixel-perfect, browser-based image compositor and pixel art editor.

What is PixelConnect?

PixelConnect is a fully client-side image compositor and pixel art tool. Every operation — compositing, editing, exporting — runs entirely in your browser. No account, no server, no install required.

It is designed around a layer-based workflow similar to Photoshop or Aseprite, but optimized for pixel art and geo-based image work.

🧱

Layer-based composition

Stack images, text, and shapes with full control over position, opacity, and z-order.

🎨

Pixel art tools

Pixelator converts any image into pixel art using palette matching and dithering algorithms.

✍️

Text & shapes

Add rasterized text with Google Fonts and geometric shapes directly to your canvas.

💾

Offline-capable

Projects auto-save to IndexedDB. Export to PNG or the native .pixcomp format.

Interface Overview

The interface has four main regions on desktop:

  • Left — Layer Panel: Lists all layers, controls visibility/lock/order, and exposes Canvas Settings at the bottom.
  • Center — Canvas: The interactive viewport where you position and preview layers.
  • Right — Property Panel: Shows position, opacity, color analysis, and editing tools for the selected layer.
  • Top — Toolbar: Project name, file operations, history, zoom, and grid controls.
Portrait / Mobile layout On narrow screens the side panels collapse. Two floating buttons (☰ bottom-left, ⚙ bottom-right) open the layers and properties drawers respectively.

First Visit

On the very first visit a demo image is loaded and an interactive guided tour starts automatically, stepping through each major area. The tour can be re-opened at any time via the ? button in the toolbar.

Quick Start

Drag one or more image files onto the canvas, or click Add in the Layer Panel to browse for files.

Click a layer in the panel or on the canvas to select it. Drag it to reposition.

Adjust position, opacity, and apply editing tools from the Property Panel on the right.

Click Save → Export PNG in the toolbar to download your composition.

Crop Canvas to Layers

The Crop to Layers button resizes the canvas to the exact bounding box of all layers. Layer positions are recalculated to remain visually identical. The viewport then auto-fits to the new canvas size.

Tip Use Crop to Layers before exporting to eliminate empty canvas space and reduce file size.

Background & Transparency

Control what appears behind all layers on the canvas.

Background Color

Click the color swatch in Canvas Settings to open the native color picker. The color fills the entire canvas and is included in PNG exports.

Transparent Background

Click the Clear button to set the background to null (transparent). Transparent pixels are preserved in PNG exports.

Checkered Background

The checkered toggle overlays an 8×8 px white/gray checkerboard to visualize transparent areas. This pattern is cosmetic only — never included in exports.

Canvas Border

Add a decorative border around the canvas area.

Enabled
Checkbox to show or hide the border. Off by default.
Color
Hex color for the border stroke. Default #000000.
Width
Thickness in pixels. The border expands outward — it does not overlap canvas content.
Opacity
0–100%. Affects only the border stroke.
Tip The border is viewport-only. It is not rendered in PNG exports.

Canvas Shadow

A soft drop shadow beneath the canvas area in the viewport.

Shadow Intensity

The slider (0–100%) controls the strength of the drop shadow. Default is 50%. Like the border, the shadow is viewport-only and does not appear in PNG exports.

Grid

A pixel-aligned grid overlay for precise placement.

Enabling the Grid

Toggle the grid with the grid icon button in the toolbar. When enabled, the grid is drawn as a canvas overlay on top of all layers.

Grid Density

The Density slider (1–8) controls how many canvas pixels fall between each grid line. A density of 1 draws a line every pixel; 4 (default) draws a line every 4 pixels.

Visibility Behavior

  • Hidden at zoom levels below 50%.
  • Between 50%–150% zoom: fades in from 0% to 30% opacity.
  • Above 150% zoom: steady 40% opacity.
  • At very low zoom, effective density is auto-multiplied to prevent overcrowding.

Persistence

Grid state and density are saved to localStorage and restored on the next visit.

Rulers & Guides

Pixel rulers and guide lines for precise layout.

Rulers

Toggle rulers from the toolbar. When enabled, horizontal and vertical pixel rulers appear along the canvas edges and track your current zoom and pan position.

Guides

Guides are stored as part of the project file. Each guide has:

axis
'x' (vertical-line guide) or 'y' (horizontal-line guide).

Unique string identifier.

Operations on Multiple Layers

  • Delete: Delete removes all selected layers (with confirmation).
  • Move: Drag any selected layer to move all together.
  • Nudge: Arrow keys move all selected layers 1 pixel.
  • Reorder: Shift+↑/↓ reorders all selected layers.
  • Copy: CtrlC copies all selected to the internal clipboard.
  • Offset: The Bulk Controls section lets you enter an X/Y offset for all selected layers.

Canvas Preview

While the editor is open, a live preview layer appears on the canvas at 70% opacity. Drag it to choose placement. Position is committed when you click Add or Update.

Shape Layers

Create pixel-perfect geometric shapes as layers.

Opening the Shape Editor

Click the ◆ button in the Layer Panel header. To edit an existing shape, click the purple ◆ icon on hover in the layer row.

Shape Types

Rectangle

Axis-aligned rectangle. Use stretch for wide/tall variants.

Circle

Ellipse. Stretch X/Y independently for ovals.

Triangle

Equilateral triangle, pointing upward.

Hexagon

Regular 6-sided polygon, flat-top.

Octagon

Regular 8-sided polygon.

Star

5-pointed star.

Shape Settings

Size
Base size in pixels. Default 200px.
Stretch X
Horizontal scale multiplier. Range 0.1–20.0.
Stretch Y
Vertical scale multiplier. Range 0.1–20.0.
Color
Hex fill color. Default #FFFFFF.

Rendering

Shapes are rendered at 1× scale. Anti-aliased pixels are thresholded: alpha ≥ 128 → fully opaque; alpha < 128 → fully transparent. Shape metadata is stored so shapes can be re-edited at any time.

Pixel-Perfect Rendering

All rendering uses imageSmoothingEnabled = false. Pixels are never interpolated — every canvas pixel maps to one or more screen pixels producing sharp pixel-art display at all zoom levels.

Panning

Navigate around large canvases.

Default Mode (Select Tool Active)

  • Left-click on a layer → selects and drags the layer.
  • Left-click on empty canvas → deselects all and pans.
  • Middle-click drag → pans the canvas.
  • Space + left-click drag → temporarily pans regardless of cursor target.

Left-Click Pan Mode

Toggle Left-Click Pan from the toolbar (hand icon button). Behavior is inverted:

  • Left-click drag → always pans.
  • Middle-click drag → drags layers.
  • Space + left-click drag → temporarily reverts to select/drag.
Tip Space temporarily toggles pan mode while held, no matter which mode is active.

Drag Info Tooltip

When dragging a layer, a tooltip near the cursor shows the live X and Y offset from the layer's original position. Toggle this in Canvas Settings via the Drag Info option.

Position & Size

Precise numerical control over a layer's position and dimensions.

Position Inputs

The X and Y inputs in the Property Panel show the top-left corner of the selected layer in canvas pixels. Values may be negative. Type a new value and press Enter or click away to apply.

While dragging, the inputs show the real-time position live.

Read-only Fields

Width
Layer image width in pixels. Use Pixelator/Resize to change.
Height
Layer image height in pixels.
Z-Index
Current stacking order. Use layer panel controls to change.
Tip Changing X or Y via the inputs pushes an undo history entry when you leave the field.

Opacity

Control the transparency of any layer.

Opacity Slider

Range 0% (fully transparent) to 100% (fully opaque). Changes apply in real time. Opacity is preserved in PNG exports.

Quick Presets

  • 50% — Half transparent for overlay/blend effects.
  • 100% — Restore full opacity.
Note Opacity is stored as a decimal (0.0–1.0). It multiplies with any per-pixel alpha in the layer's image data.

Color Analysis

On-demand pixel frequency analysis of the selected layer.

Modes

Analyze RGB

Counts pixels by RGB values. Ignores alpha — transparent pixels are counted at their solid color.

Analyze RGBA

Counts pixels by RGBA. Fully transparent pixels (alpha = 0) are excluded entirely.

Results

A virtually-scrolled list sorted by pixel count, showing: color swatch, hex code, pixel count, and percentage of total pixels.

Click Unload to free memory. Results clear automatically when you select a different layer.

Building a Custom Palette

  • Type hex codes directly into the palette input.
  • Eyedropper: Click pixels on the image to pick colors. Supports zoom/pan.
  • Gradient Picker: Draw a line across the image; sampled colors along the line are added. Useful for gradients.
  • GeoPixels Lookup: Enter a GeoPixels user ID to import their unlocked palette.
  • K-means Auto-Palette: Auto-selects the N most representative colors via k-means clustering.
  • Suggest Colors: Analyzes image and suggests top N visually distinct colors. Toggle "prefer distinct colors" for broader coverage.
  • Filter Trivial Colors: Remove palette entries below a pixel-count or percentage threshold.

Before/After Comparison

A drag slider reveals before/after side-by-side within the preview area.

Auto-Update

When enabled, the preview regenerates automatically as settings change. Disable for performance with large images.

Crop

Crop a layer to a sub-rectangle. Open via Property Panel → Modify → Crop.

Crop Interface

The modal shows the layer image with a moveable, resizable crop rectangle.

  • Drag the rectangle to move the entire crop selection.
  • Drag any of the 8 handles (corners + mid-edges) to resize.
  • Zoom in/out for pixel-level precision.

Applying

Click Apply to commit. The layer's imageData, x, y, width, and height are all updated. If this is the only layer, the canvas auto-crops to the new bounds.

Background Removal

Remove backgrounds using AI or manual tools. Open via Property Panel → Modify → Background Removal.

AI Mode

Uses the RMBG-1.4 neural network (ONNX Runtime, runs 100% in the browser — no data sent to a server).

  • First use downloads the model (~170 MB) with a progress bar. Cached in the browser after that.
  • Best for photos with clear subject/background separation.

Click Mode (Flood Fill)

Click a pixel — the tool flood-fills connected similarly-colored pixels to transparent.

  • Tolerance slider: How far the color can differ before the fill stops. Default: 30. Higher = removes more area.

Brush Mode

Paint over the image to erase pixels. Has a configurable brush radius and its own local undo/redo stack.

Lasso Mode

Draw a freehand polygon; the interior is erased on mouse release.

Common Features

  • All modes work on a working copy — original is preserved until you click Apply.
  • Reset reverts to the original image.
  • Canvas shows a checkerboard to visualize transparency.
  • Each mode has its own local undo/redo.

Transparency Mask

Remove low-alpha pixels based on a threshold. Open via Property Panel → Modify → Transparency Mask.

How It Works

Any pixel whose alpha value is at or below the threshold becomes fully transparent. Pixels above the threshold keep their original color and alpha.

Threshold Slider

Range 0–255. A threshold of 0 removes only fully-transparent pixels. A threshold of 128 removes all pixels less than 50% opaque.

Use Template Palette

When enabled, the image is also quantized to the GeoPixels palette while the mask is applied — useful for palette-conformant sprites in one step.

Applying

Click Apply to write the result to the layer. If this is the only layer, the canvas auto-crops to the visible bounding box.

Exporting

Save your work as a PNG image or in the native .pixcomp format.

Export PNG

Save → Export PNG. Composites all visible layers in z-order, applies opacities, and includes the background color (if set).

Scale
1× = pixel-for-pixel. 2× doubles both dimensions. Default: 1×.
Filename
Auto-named {ProjectName}_{scale}x.png.
Tip Canvas border and shadow are NOT rendered in exports — only pixels within the canvas bounds.

Export .pixcomp

Save → Export .pixcomp file. Saves the complete project as a JSON file with all layer image data embedded as Base64. Filename: {ProjectName}.pixcomp.

Importing

Load projects or images into PixelConnect.

Open File

Load → Open .pixcomp / image file. Accepts:

  • .pixcomp — Restores a full PixelConnect project (all layers, canvas settings, rulers, grid).
  • PNG, JPG, GIF, BMP, WebP — Each image becomes a new layer.

Drag & Drop

Drag image files onto the canvas. Supported: PNG, JPG, GIF, WebP, SVG, BMP.

Clipboard Paste

CtrlV pastes an image as a new layer, or plain text as a rasterized text layer.

New Project

The New Project button resets to a blank 3276×3276 canvas. If unsaved changes exist you will be prompted to confirm.

Browser Storage

Save and load named projects directly in the browser using IndexedDB.

Save to Browser

Save → Save to Browser stores the current project with a thumbnail to IndexedDB. Creates a named, persistent snapshot.

Load from Browser

Load → Load from Browser opens the Saved Projects Manager, listing all saved projects with thumbnail, name, save date, file size, layer count, and canvas dimensions. From here you can Load, Rename, or Delete any project.

Auto-Save Cache

Separately, PixelConnect auto-saves the current project state to IndexedDB 2 seconds after any change. This cache is auto-restored on the next page load — protecting against accidental tab closes or crashes.

Warning Browser storage is limited by your browser's quota. For long-term or cross-device storage, always export a .pixcomp file.

Undo & Redo

Step through editing history to correct mistakes or revisit earlier states.

Controls

The ↶ undo and ↷ redo buttons in the toolbar show tooltip counts of available steps. Keyboard: CtrlZ / CtrlShiftZ or CtrlY.

What Is Tracked

History captures the full project state: all layer data, canvas settings, grid, rulers. The viewport (zoom/pan) is intentionally excluded so it is preserved across undo/redo.

History Limits

  • Maximum 50 steps retained. Older steps are discarded at the limit.
  • History is in-memory only — not persisted to disk or .pixcomp files.

Auto-History

Changes are debounced 500ms before committing to history. Rapid edits naturally group into single history steps. A layer drag commits one history entry for the entire move.

Auto-Save

Your project is continuously and automatically saved to the browser.

How It Works

PixelConnect saves the current project to IndexedDB 2 seconds after any change. On the next page load the auto-saved state is automatically restored — protecting against tab closes or crashes.

Exclusions

Preview layers from the Text and Shape editors (__text_canvas_preview__, __shape_canvas_preview__) are stripped before saving.

Dirty State

When there are unsaved changes the tab title is prefixed with ● (e.g. ● MyProject - PixelConnect). The browser beforeunload warning fires if you try to close the tab. The dirty flag clears when you Save to Browser or Export .pixcomp.

Debug Menu

A developer tool showing the full internal history stack.

Accessing

Press CtrlShiftD to open the Debug History Modal.

Contents

Shows the complete past history stack, current project state, and redo future stack. Useful for diagnosing unexpected undo behavior.

Developer use only This modal is not part of the normal user workflow.
